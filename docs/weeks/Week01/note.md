
- 정적 분석 도구 사용 설정
    - clang-tidy
    - 피곤하거나 생각이 둔해지는 경우 규칙을 지키기 어려우며, 매번 생각하고 있으려면 힘듦 -> 자동으로 검사해주는 도구 도입

    - Google 베이스
    항목,설명
bugprone-*,"논리적 오류가 발생하기 쉬운 코드 패턴(메모리 누수 위험, 잘못된 연산자 사용 등)을 찾아냅니다."
google-*,"Google C++ 스타일 가이드에 특화된 규칙들입니다. (예: 헤더 가드 형식, 캐스팅 방식 등)"
modernize-*,"NULL 대신 nullptr 사용, auto 사용 권장 등 최신 C++(C++11 이상) 문법으로 전환을 제안합니다."
performance-*,불필요한 객체 복사나 비효율적인 루프 등 성능 저하 요소를 감지합니다.
readability-*,코드의 가독성을 높이는 규칙들입니다. 네이밍 규칙도 여기에 포함됩니다.
