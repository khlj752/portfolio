# VS Code C++ Google Style 설정 가이드

이 문서는 **Google C++ 스타일 가이드**를 준수하기 위해 VS Code 환경에서 `.clang-format` 및 `.clang-tidy`를 설정하는 방법을 안내합니다.

---

## 1. 코드 포맷 설정 (`.clang-format`)
프로젝트 루트 폴더에 `.clang-format` 파일을 생성하고 아래 내용을 입력하세요.

```yaml
# .clang-format
BasedOnStyle: Google
IndentWidth: 2
ColumnLimit: 80
PointerAlignment: Left
UseTab: Never
NamespaceIndentation: None
AccessModifierOffset: -2
BreakBeforeBraces: Attach
Standard: Latest
SortIncludes: true
AllowShortFunctionsOnASingleLine: Empty
```


## 2. 네이밍 규칙 및 정적 분석 (.clang_tidy)

``` yml
# .clang-tidy
Checks: >
  -*,
  bugprone-*,
  google-*,
  misc-*,
  modernize-*,
  performance-*,
  readability-*,
  -readability-braces-around-statements,
  readability-identifier-naming

WarningsAsErrors: 'readability-identifier-naming,google-*'

CheckOptions:
  - { key: readability-identifier-naming.ClassCase,           value: PascalCase }
  - { key: readability-identifier-naming.StructCase,          value: PascalCase }
  - { key: readability-identifier-naming.FunctionCase,        value: PascalCase }
  - { key: readability-identifier-naming.VariableCase,        value: snake_case }
  - { key: readability-identifier-naming.ParameterCase,       value: snake_case }
  - { key: readability-identifier-naming.ConstantCase,        value: PascalCase }
  - { key: readability-identifier-naming.ConstantPrefix,      value: 'k' }
  - { key: readability-identifier-naming.PrivateMemberCase,   value: snake_case }
  - { key: readability-identifier-naming.PrivateMemberSuffix, value: '_' }
  - { key: readability-identifier-naming.NamespaceCase,       value: snake_case }
  - { key: readability-identifier-naming.MacroDefinitionCase, value: UPPER_CASE }
```

### 3. 정적 분석 체크 항목 설명

위 설정에서 활성화된 각 정적 분석 모듈의 역할은 다음과 같습니다.

*   **`bugprone-*`** (버그 유발 가능성 탐지)
    *   코드에서 잠재적인 버그를 일으킬 수 있는 패턴을 예방적으로 찾아냅니다.
    *   예: 잘못된 형변환, 논리 오류, 안전하지 않은 함수 사용 등.

*   **`google-*`** (Google 스타일 가이드 준수)
    *   Google C++ 코딩 표준을 위반하는 패턴을 검사합니다.
    *   예: 네이밍 규칙, 생성자 명시적 호출(`explicit`), 글로벌 네임스페이스 오염 방지 등.

*   **`misc-*`** (기타 유용한 검사)
    *   다른 카테고리에 속하지 않지만 코드 품질을 높이는 다양한 검사를 포함합니다.
    *   예: 사용되지 않는 인자, 비효율적인 연산 등.

*   **`modernize-*`** (최신 C++ 문법 적용)
    *   오래된 C++ 스타일(Legacy C++)을 C++11/14/17/20 등 최신 문법(Modern C++)으로 개선하도록 제안합니다.
    *   예: `auto` 사용 권장, `nullptr` 사용, `override` 키워드 명시 등.

*   **`performance-*`** (성능 최적화)
    *   불필요한 복사나 비효율적인 메모리 사용 등 성능 저하를 유발할 수 있는 코드를 탐지합니다.
    *   예: `const reference` 대신 값 복사로 전달하는 경우 수정 제안.

*   **`readability-*`** (가독성 향상)
    *   코드를 읽고 이해하기 쉽게 만드는 규칙을 검사합니다.
    *   예: 식별자 이름 규칙(네이밍), 복잡한 표현식 단순화, 불필요한 괄호 제거 등.
    *   **`readability-identifier-naming`**: 변수, 함수, 클래스 등의 이름이 지정된 규칙(CheckOptions)을 따르는지 엄격하게 검사합니다.
