# 개발 일정 및 마일스톤 (Project Roadmap)

## 목표
- **최종 산출물**: DirectX 12 자체 엔진 기반 3D Horde Survivor 게임 스팀 출시
- **포트폴리오**: 개발 과정의 고민과 해결책이 담긴 기술 문서 및 DevLog

## 전체 일정 (Estimated: 9-10 Weeks)

### Milestone 1: The Foundation (기반 다지기) - 2 Weeks
**목표**: 검은 화면에 삼각형을 띄우고, 기본적인 게임 루프를 돌린다.
- [ ] **Week 1**: 
    - [x] Win32 Window 클래스 캡슐화
    - [ ] DX12 Device, CommandQueue, SwapChain 초기화
    - [ ] 화면 Clear 및 프레임 버퍼 스왑
- [ ] **Week 2**: 
    - [ ] Vertex Buffer, Index Buffer 생성
    - [ ] Shader (HLSL) 컴파일 및 파이프라인 상태 객체(PSO) 생성
    - [ ] 삼각형 그리기 성공 (The "Hello World" of Graphics)

### Milestone 2: 3D Visualization (3D 시각화) - 2 Weeks
**목표**: 3D 공간감을 구현하고 카메라로 돌아다닌다.
- [ ] **Week 3**: 
    - [ ] 수학 라이브러리(행렬/벡터) 통합 또는 구현
    - [ ] Constant Buffer를 통한 World-View-Projection 행렬 전달
    - [ ] Depth Stencil Buffer 설정 (3D 깊이 처리)
- [ ] **Week 4**: 
    - [ ] 텍스처 로딩 및 매핑
    - [ ] 카메라 클래스 구현 (키보드/마우스 입력 연동)

### Milestone 3: The Horde Technology (대규모 물량 처리) - 2 Weeks
**목표**: 1만 개의 적을 프레임 저하 없이 렌더링한다. (엔진의 핵심 차별점)
- [ ] **Week 5**: 
    - [ ] **Instancing Rendering** 구현 (단일 Draw Call로 수천 개 그리기)
    - [ ] 구조적 버퍼(Structured Buffer)를 이용한 데이터 전달
- [ ] **Week 6**: 
    - [ ] **Data-Oriented Design**: 적들의 데이터를 SoA(Structure of Arrays) 형태로 전환
    - [ ] 간단한 이동 AI 구현 및 성능 프로파일링

### Milestone 4: Combat & Spark (전투 및 이펙트) - 2 Weeks
**목표**: 화려한 전투를 구현하고 최적화한다.
- [ ] **Week 7**: 
    - [ ] **Spatial Partitioning**: Grid 또는 QuadTree를 이용한 충돌 최적화
    - [ ] 플레이어 무기 및 사격 메커니즘
- [ ] **Week 8**: 
    - [ ] **GPU Particles**: 컴퓨트 쉐이더(Compute Shader)를 활용한 대량 파티클 시스템
    - [ ] 피격 효과 및 사운드 연동 (FMOD 등 활용)

### Milestone 5: Polish & Release (마무리 및 출시) - 2 Weeks
**목표**: 게임을 완성하고 세상에 내놓는다.
- [ ] **Week 9**: 
    - [ ] 게임 루프 완성 (Start -> Play -> GameOver -> Restart)
    - [ ] UI (ImGui 활용) 및 HUD 구현
    - [ ] 직렬화(Save/Load)
- [ ] **Week 10**: 
    - [ ] 스팀 빌드 패키징
    - [ ] 최종 버그 수정 및 최적화
    - [ ] 포트폴리오 문서 정리 및 데모 영상 제작

## Daily Routine
1.  **Plan**: 오늘 구현할 기능의 설계 의도(Why) 정리.
2.  **Code**: 집중 코딩.
3.  **Log**: `docs/dev_log.md`에 문제 해결 과정 기록.
